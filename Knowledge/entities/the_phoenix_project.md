# The Phoenix Project

## Summary
The definitive 2013 novel by Gene Kim, Kevin Behr, and George Spafford introducing DevOps to the mainstream. It uses a fictional narrative about an IT manager (Bill) saving a failing company (Parts Unlimited) to explain Lean principles applied to IT. 

## Relevance
- **Domain alignment:** Core text for the Manager Partnership and Continuous Improvement pillars. Gives vocabulary to the "Decision Fog" and wait-state problems.
- **Source of frameworks:** Specifically validates Kaan's focus on "Unplanned Work" (reducing it from 28% to 12%).
